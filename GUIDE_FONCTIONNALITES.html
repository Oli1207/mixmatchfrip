La vitrine de la marque. Premier point de contact avec la cliente.

Où

Ce qu'elle contient

L'interface de découverte de tous les produits disponibles, avec filtres multi-critères en temps réel.

Où

/catalogue · /nouveautes (redirige vers /catalogue?sort=recent) · /soldes (redirige vers /catalogue?sort=price_asc)

Filtres disponibles

FiltreValeurs possiblesComportement

Comment ça fonctionne

  • La visiteuse arrive sur /catalogue. Tous les produits disponibles s'affichent.
  • Elle sélectionne un ou plusieurs filtres dans le panneau latéral. Chaque changement déclenche un appel API immédiat — pas besoin de valider.
  • Les filtres actifs sont persistés dans l'URL (query string), donc la page est partageable.
  • Un compteur affiche le nombre de résultats trouvés.
  • Elle clique sur une carte produit pour accéder à la fiche détaillée.
Note : Les produits marqués is_available = False ou stock = 0 n'apparaissent pas dans le catalogue public.

Boutique publique

Fiche produit

La page la plus importante de la conversion. Elle présente le produit sous toutes ses coutures et déclenche l'achat.

Où

/product/{slug} — ex : /product/zara-robe-midi-fluide

Contenu de la fiche

📸 Galerie d'images

Image principale cliquable + miniatures. Lightbox plein écran au clic. Navigation par flèches ou clic sur thumbnail.

🏷️ Infos produit

Nom, marque, badge état (Neuf/Excellent/…), prix barré + prix réduit si en solde, taille standard + taille étiquette.

📐 Mesures à plat

Tableau des mensurations (épaule, poitrine, taille, hanches, longueur, manches) en centimètres. Seules les mesures renseignées s'affichent.

💡 Conseils Mix & Match

3 idées de tenues générées par la vendeuse. S'affichent sous forme de liste pour inspirer la cliente.

🛒 Ajout au panier

Bouton « Ajouter au panier » + bouton « Acheter maintenant » (ajoute et redirige directement vers le checkout).

❤️ Wishlist

Icône cœur : toggle ajout/retrait de la liste de souhaits. Nécessite d'être connectée. Si non connectée, invite à se connecter.

Processus de création d'un produit (admin → boutique)

  • Dans le tableau de bord admin, aller dans Produits → Ajouter.
  • Renseigner : nom, marque, catégorie, sous-catégorie, état, taille, prix (et prix original si en solde).
  • Ajouter les mesures à plat (facultatif mais fortement recommandé).
  • Écrire la description, les 4 arguments de vente, les idées Mix & Match et le conseil expert.
  • Uploader les photos (la première uploadée devient la photo principale, réordonnables).
  • Sauvegarder → le produit est immédiatement visible sur le catalogue.
Slug automatique : Le slug URL est généré automatiquement depuis le nom et la marque (zara-robe-midi-fluide). Il ne change pas si on modifie le nom pour éviter les liens cassés.

Boutique publique

Panier

Gestion des articles sélectionnés avant l'achat. Fonctionne pour les visiteuses non connectées grâce à un identifiant local.

Où

/cart · Icône panier dans la barre de navigation (avec badge compteur)

Fonctionnement technique

Le panier est stocké côté serveur (base de données), identifié par un cart_id UUID persisté dans le localStorage du navigateur. Cela permet :

  • D'utiliser le panier sans créer de compte.
  • De retrouver son panier si on ferme et rouvre le navigateur.
  • Lors d'une connexion, le panier anonyme est fusionné avec le panier du compte existant.

Boutique publique

Processus de commande

Tunnel de conversion en 3 étapes : livraison → paiement → confirmation. Conçu pour minimiser l'abandon.

Où

/checkout

Étape 1 — Adresse de livraison

  • La cliente saisit son prénom, nom, email, téléphone, adresse complète, ville, province et code postal.
  • Dès que le code postal est saisi (6 caractères), les tarifs de livraison sont automatiquement récupérés via Chit Chats. Un délai de 600 ms évite les appels inutiles pendant la frappe.
  • Les options de livraison s'affichent avec leur délai estimé et leur prix. La première option est pré-sélectionnée.
  • Si la cliente est connectée, ses informations sont pré-remplies depuis son profil.
  • Option « Créer un compte » pour les invités — crée automatiquement un compte avec les infos saisies.

Étape 2 — Paiement

  • Récapitulatif de la commande avec les articles, sous-total, remise promo éventuelle, frais de port et total.
  • Champ code promo avec validation en temps réel.
  • Au clic sur « Payer », la commande est créée en base, puis une session Stripe est générée.
  • La cliente est redirigée vers la page Stripe (sécurisé, certifié PCI-DSS). Ses données bancaires ne transitent jamais par nos serveurs.

Étape 3 — Confirmation

Après paiement Stripe réussi, la cliente revient sur /payment-success/{numéro}. La commande est confirmée, l'envoi Chit Chats est créé, l'email de confirmation est envoyé.

Logistique & Paiement

Stripe & Chit Chats

Paiement sécurisé via Stripe et création automatique de l'envoi postal via Chit Chats après confirmation.

Flux de paiement Stripe

  • Création commande : La commande est enregistrée en base avec le statut pending.
  • Session Stripe : Le backend crée une Checkout Session Stripe avec le montant exact, les articles et les URLs de retour.
  • Redirection : La cliente est envoyée sur la page Stripe (paiement par carte, Apple Pay, Google Pay…).
  • Retour succès : Stripe redirige vers /payment-success/{order_number}?session_id=…. Le backend vérifie auprès de Stripe que le paiement est bien paid avant de confirmer.
  • Retour échec : Stripe redirige vers /payment-failed/{order_number}. La commande reste en pending, la cliente peut réessayer.

Flux Chit Chats (livraison)

  • Calcul des tarifs : Au moment de la saisie du code postal, un shipment temporaire est créé dans Chit Chats pour obtenir les prix de livraison, puis immédiatement supprimé. Aucun frais n'est engagé à ce stade.
  • Après paiement confirmé : Un shipment permanent est créé dans Chit Chats avec les vraies informations de la commande (adresse, poids, valeur déclarée, méthode choisie).
  • Shipment ID : L'identifiant Chit Chats est sauvegardé sur la commande en base (shipment_id).
  • Email de confirmation : L'URL de suivi Chit Chats est incluse dans l'email. La cliente reçoit un bouton cliquable pour suivre son colis dès que le numéro est disponible.

Marketing

Codes promo & Newsletter

Outils marketing pour fidéliser, récompenser et maintenir le lien avec la communauté.

Codes promo — comment les créer

  • Dans le tableau de bord admin : Codes promo → Nouveau.
  • Saisir le code (ex : BIENVENUE10), choisir le type : pourcentage ou montant fixe.
  • Définir la valeur de remise, le montant minimum de panier (optionnel) et la limite d'utilisation (optionnel, laisser vide = illimité).
  • Activer le code. Il est immédiatement utilisable en checkout.

Codes promo — comment les utiliser (côté cliente)

  • À l'étape 2 du checkout (paiement), un champ « Code promo » est affiché.
  • La cliente saisit son code et clique « Appliquer ».
  • Le serveur vérifie : code valide, actif, sous le plafond d'utilisation, montant minimum respecté.
  • La remise s'affiche instantanément sur le récapitulatif. Le compteur d'utilisation s'incrémente à la validation de la commande.

Newsletter — inscription

Un formulaire d'inscription est présent dans le footer et sur la page d'accueil. La saisie d'un email valide ajoute l'adresse à la liste des abonnées. Un message de confirmation s'affiche immédiatement.

Newsletter — gestion admin

Dans le tableau de bord : Newsletter → liste complète des abonnées avec date d'inscription. Possibilité de supprimer une entrée (désinscription manuelle). Export CSV possible pour envoi via tout outil d'emailing.

Administration

Créer & gérer un produit

Guide pas-à-pas pour mettre un vêtement en ligne, de la photo à la publication.

Étapes de mise en ligne d'un vêtement

  • Photos : Prendre les photos (fond clair, lumière naturelle recommandée). Minimum 1 photo, idéalement 3–5 vues différentes.
  • Nouveau produit : Dans le dashboard → Produits → Bouton « + Nouveau produit ».
  • Informations de base : Nom du vêtement, marque, catégorie, sous-catégorie, taille (menu + étiquette physique), état, poids en grammes.
  • Prix : Prix de vente. Si c'est une réduction, renseigner aussi le prix original (barré). Le pourcentage de remise est calculé automatiquement.
  • Mesures à plat : Mesurer le vêtement à plat (en cm) : épaules, poitrine (aisselle à aisselle × 2), taille, hanches, longueur, manches. Ces mesures sont plus fiables que les tailles standards.
  • Contenu marketing : 4 arguments de vente courts (bullet points), description longue, matière, détails de coupe, 3 idées de tenues Mix & Match, conseil expert.
  • Upload photos : Glisser-déposer les photos. Définir la photo principale. L'ordre est modifiable.
  • Publier : Cocher « Disponible » et sauvegarder. Le produit apparaît immédiatement sur le catalogue.

Ce que le dashboard affiche

📊 5 KPIs clés

Sessions totales · Pages vues · Achats · Taux de conversion · Vues produit. Filtrables sur 7j / 14j / 30j / 90j.

📈 Courbe sessions/pages vues

Graphique linéaire jour par jour sur la période sélectionnée. Permet de voir les pics d'activité.

🌍 Sources de trafic

D'où viennent les visiteuses : direct, Instagram, Google, email, lien partagé… Chaque source UTM est identifiée.

🎯 Tunnel de conversion

Visualisation du chemin : Vues produit → Ajout panier → Début checkout → Achat. Identifie où les gens abandonnent.

📱 Répartition appareils

Mobile / Tablette / Desktop. Permet de prioriser les efforts d'optimisation.

📣 Campagnes UTM

Tableau des campagnes avec source, sessions et achats générés. Savoir quelle pub rapporte.

Utiliser les paramètres UTM (pour les publications Instagram/ads)

  • Construire un lien avec les paramètres UTM, ex : https://mixmatchfrip.com/?utm_source=instagram&utm_medium=story&utm_campaign=soldes_ete_2026
  • Publier ce lien dans la bio Instagram, les stories, les publicités Meta, les emails…
  • Dans le dashboard analytiques, la section « Campagnes UTM » montre combien de sessions et d'achats ce lien a générés.
  • Comparer les campagnes pour identifier les canaux les plus rentables.
Respect de la vie privée : Les sessions sont identifiées par un UUID anonyme stocké dans le localStorage. Aucun cookie de tracking tiers. Aucune donnée personnelle dans les événements. Conforme à la loi 25 (Québec) et au RGPD.

Design mobile

Le site est entièrement responsive avec des points de rupture optimisés à 768px, 640px et 480px. Points clés :

  • Cibles tactiles : Tous les boutons font minimum 44px de hauteur (standard Apple / Google).
  • Pas de zoom iOS : Les champs de formulaire ont une taille de police ≥ 16px pour éviter le zoom automatique sur iPhone.
  • Encoche iPhone : Le site respecte la safe area (notch et home indicator) via env(safe-area-inset-*).
  • Navigation mobile : Menu hamburger avec drawer animé. Fermeture au tap en dehors.

Résumé des technologies

Stack technique

Frontend — React 19

Vite · React Router · Zustand (état global) · Axios · React Helmet Async (SEO) · Recharts (graphiques analytics) · Bootstrap + CSS custom

Backend — Django 5 + DRF

Django REST Framework · SimpleJWT (auth tokens) · PostgreSQL · Stripe SDK · API Chit Chats

Paiement

Stripe Checkout (hébergé) — PCI-DSS compliant. Webhooks pour confirmation asynchrone.

Chit Chats API pour les tarifs de livraison en temps réel et la création automatique des envois.

Email

Django email avec templates HTML/TXT. Confirmation de commande avec lien de suivi Chit Chats.

Analytics maison

Sessions UUID + événements e-commerce en base de données. Dashboard admin avec graphiques recharts. Aucune dépendance externe.
